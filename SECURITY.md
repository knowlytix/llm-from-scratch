# Security Policy

## Reporting a vulnerability

**Please do not report security vulnerabilities through public GitHub issues.**

Instead, email **security@knowlytix.ai** with:

- a description of the issue and its impact,
- steps to reproduce (a minimal proof-of-concept if possible), and
- any suggested remediation.

You can expect an acknowledgement within a few business days. We will keep you
informed as we investigate and work on a fix, and we will credit you in the
release notes unless you prefer to remain anonymous.

## Scope

This policy covers the open-source library in this repository. It runs without
the optional, licensed GMS backend (`knowlytix`); vulnerabilities specific to
that licensed component should also be sent to **security@knowlytix.ai**.

## Supported versions

This project is pre-1.0 and under active development. Security fixes are applied
to the latest released version on the default branch.
